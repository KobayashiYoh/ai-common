description: PBI情報を元に実装方針を作成する。
---

以下の手順で実装方針を作成してください：

## 1. PBI情報の確認

- 直前に `/fetch-issue` で取得したPBI情報を使用する
- PBI情報がない場合は、ユーザーにPBI IDを確認して `/fetch-issue` を実行する

## 2. 既存コードベースの調査

実装に関連する既存コードを調査：

- PBIの内容から影響を受ける可能性のあるファイル・モジュールを特定
- 関連する既存の実装パターンを確認
- 既存のテストコードを確認

## 3. 実装方針の作成

PBI情報と既存コードベースの調査結果を踏まえて、以下の2点で実装方針を作成：

### 3.1 実装アプローチ

- 既存のコードベースを考慮した実装方針
- 影響範囲の特定
- 必要な変更箇所の洗い出し
- 実装の順序や依存関係

### 3.2 実装タスク

- 実装を細分化したタスクリスト
- 各タスクの依存関係
- 優先順位

## 4. 出力形式

以下の構成でClaude Codeの通常の返答として表示：

```
# 実装方針

## 対象PBI
- PBI番号: [ID]
- Title: [タイトル]

## 実装アプローチ
[実装方針の詳細]
- 影響範囲
- 変更箇所
- 実装順序

## 実装タスク
1. [タスク1]
2. [タスク2]
3. [タスク3]
...

（各タスクの依存関係や優先順位も記載）
```

## 注意事項

- PBI情報がコンテキストに存在しない場合は、まず `/fetch-issue` の実行を促す
- 実装タスクは具体的で実行可能な単位に分割する
- 各タスクの依存関係を明確にする
- 必要に応じて関連する既存コードを調査する
